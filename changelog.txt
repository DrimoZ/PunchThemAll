PunchThemAll changelog
======================

Unreleased
----------

### Configuration

* Replaced the original flat common config with modular sections:
  `Interactions`, `Players`, `Drops`, `Loader`, and `Debug`.
* Added global runtime gates for enabling/disabling all interactions, left-click
  interactions, right-click interactions, block targets, air targets, fluid
  targets, and transformations.
* Replaced the cooldown key with `cooldown_ticks` and added support for `0` to
  disable cooldowns.
* Added `max_matches_per_click` to cap the number of interactions processed from
  one click.
* Added fake-player controls for enabling automation, applying cooldowns to fake
  players, and applying player-only side effects to fake players.
* Added controls for player damage and food consumption effects.
* Added separate real-player and fake-player inventory insertion controls.
* Added configurable world-drop offset and velocity.
* Added loader controls for recursive discovery, fail-fast mode, and generated
  ID lowercasing.
* Added debug logging controls for loaded interactions and skipped runtime
  interactions.

### Interaction loading

* Interaction JSON files are loaded deterministically by relative path.
* Interaction JSON files can be organized recursively below
  `config/punchthemall/interactions`.
* Interaction IDs are generated from relative paths, for example
  `interactions/early_game/flint.json` becomes `pta:early_game/flint`.
* Invalid interaction files now log the source path and can optionally stop the
  loader when `fail_fast` is enabled.

### Gameplay behavior

* Runtime interaction processing now honors the new config gates.
* Fake-player behavior is more configurable for automation-heavy packs.
* Transformations can be disabled globally without disabling drops.
* Drops use configurable insertion and world-spawn behavior.
* Direction fallback was improved to avoid null direction handling in edge cases.

### JEI

* JEI click tooltips now show the interaction ID.
* Block-state blacklist tooltips now display blacklist entries correctly.
* Unknown enchantment IDs in NBT tooltips no longer crash tooltip generation.
* Out-of-range enchantment levels fall back to numeric display instead of
  throwing during Roman numeral formatting.

### Documentation

* Replaced the Forge MDK README with a PunchThemAll-focused README.
* Added `docs/configuration.md` with all config sections, defaults, examples,
  presets, and migration notes.
* Added `docs/interactions.md` with loader behavior, reload notes, runtime gates,
  JEI display details, and pack organization tips.
